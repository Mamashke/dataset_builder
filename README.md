# dataset_builder

Инструмент командной строки для автоматического построения обучающих датасетов для YOLO-моделей. Берёт сырые видеозаписи, превращает их в размеченный, сбалансированный и аугментированный датасет, готовый к обучению.

---

## О проекте

**Цель** — универсальный конструктор обучающей выборки: от исходного видео до готового датасета в один CLI-интерфейс.

**Демонстрационная задача** — обнаружение людей с БПЛА в сложных условиях съёмки (дождь, туман, низкая яркость). Проект поддерживает два источника данных:
- `real` — записи с реальной камеры
- `airsim` — записи из симулятора AirSim

**Поддерживаемые форматы экспорта:** YOLO (`data.yaml`) и COCO (`annotations.json`).

---

## Архитектура

Пайплайн состоит из пяти последовательных шагов, каждый реализован в отдельном модуле:

```
video → [loader] → frames → [annotator] → annotations
                                ↓
                          [augmentor] → augmented frames
                                ↓
                          [balancer] → dataset/
                                ↓
                          [exporter] → data.yaml / annotations.json
```

### `modules/loader.py`
Извлекает кадры из MP4-видеофайлов. Интерактивно предлагает выбрать файлы для обработки, сохраняет каждый N-й кадр (по умолчанию каждый 5-й, ~6 fps при 30 fps видео) и приводит их к целевому разрешению (640×640 по умолчанию).

### `modules/annotator.py`
Размечает кадры с помощью предобученной модели YOLOv8. Режимы работы:
- **auto** — автоматический инференс, создаёт YOLO-формат txt для каждого кадра
- **manual** — показывает список кадров без аннотаций (для ручной разметки сторонними инструментами)

### `modules/augmentor.py`
Применяет пиксельные аугментации к оригинальным кадрам, создавая дополнительные обучающие примеры. Поддерживает 5 типов эффектов; разметка (txt-файлы) копируется к каждой аугментированной копии без изменений, так как геометрия объектов не меняется.

### `modules/balancer.py`
Собирает финальный датасет в три шага:
1. **Фильтрация** — отбрасывает слишком тёмные и размытые кадры
2. **Балансировка** — приводит соотношение позитивных/негативных примеров к целевому (1:3 по умолчанию)
3. **Сборка** — копирует отобранные кадры и метки в `dataset/images/` и `dataset/labels/`

### `modules/exporter.py`
Создаёт конфигурационные файлы для обучения:
- **YOLO** — `data.yaml` с путями и именами классов
- **COCO** — `annotations.json` с конвертацией координат из нормализованного формата YOLO в абсолютные пиксели

### `modules/project.py`
Управляет изолированными проектами: создание, загрузка, список, удаление. Каждый проект хранит свои данные в `projects/{name}/` и метаданные в `project.json` (текущий шаг, статистика, точка старта пайплайна).

### `modules/logger.py`
Единая настройка логирования. При каждом запуске создаёт один лог-файл `projects/{name}/logs/run_YYYYMMDD_HHMMSS.log`, куда пишут все модули через корневой логгер Python. Вывод дублируется в консоль.

## Установка

```bash
pip install -r requirements.txt
pip install ultralytics   # для авторазметки через YOLOv8
```

`requirements.txt` содержит `opencv-python>=4.13.0`. Библиотека `numpy` устанавливается автоматически как зависимость OpenCV.

> **Python 3.8+**, проверено на Windows 11 и Ubuntu 22.04.

## Быстрый старт

### Сценарий 1 — Есть видео

```bash
# 1. Создать проект (выбрать пункт "1. У меня есть видео")
python main.py --new-project drones

# 2. Скопировать видео в projects/drones/raw/real/

# 3. Запустить весь пайплайн
python main.py --project drones --all
```

Пайплайн последовательно спросит: какие файлы загрузить → какой источник размечать → режим разметки → формат экспорта.

---

### Сценарий 2 — Есть кадры без разметки

```bash
# 1. Создать проект (выбрать пункт "2. У меня есть кадры без разметки")
python main.py --new-project drones

# 2. Скопировать кадры в projects/drones/frames/real/

# 3. Запустить начиная с разметки
python main.py --project drones --all
# или явно указать стартовый шаг:
python main.py --project drones --all --from annotate
```

---

### Сценарий 3 — Есть размеченный датасет

```bash
# 1. Создать проект (выбрать пункт "3. У меня есть размеченный датасет")
python main.py --new-project drones

# 2. Скопировать:
#    кадры → projects/drones/frames/real/
#    метки → projects/drones/annotations/real/

# 3. Запустить балансировку и экспорт
python main.py --project drones --all
# или только нужные шаги:
python main.py --project drones --balance
python main.py --project drones --export --format yolo
```

## Команды

### Управление проектами

| Команда | Описание |
|---|---|
| `--new-project НАЗВАНИЕ` | Создать новый проект (интерактивно) |
| `--list-projects` | Показать все проекты и их статус |
| `--delete-project НАЗВАНИЕ` | Удалить проект (с подтверждением) |

```bash
python main.py --new-project bpla_dataset
python main.py --list-projects
python main.py --delete-project test_video
```

---

### Запуск пайплайна

Все команды пайплайна требуют `--project`.

#### Запуск всего пайплайна

```bash
# С шага, сохранённого в project.json (start_from)
python main.py --project bpla --all

# С указанного шага (переопределяет project.json)
python main.py --project bpla --all --from load
python main.py --project bpla --all --from annotate
python main.py --project bpla --all --from augment
python main.py --project bpla --all --from balance
python main.py --project bpla --all --from export
```

Допустимые значения `--from`: `load`, `annotate`, `augment`, `balance`, `export`.

#### Запуск отдельных шагов

```bash
python main.py --project bpla --load       # только загрузка видео
python main.py --project bpla --annotate   # только разметка
python main.py --project bpla --augment    # только аугментация
python main.py --project bpla --balance    # только балансировка

python main.py --project bpla --export              # экспорт (спросит формат)
python main.py --project bpla --export --format yolo
python main.py --project bpla --export --format coco
```

---

### Очистка данных

```bash
# Интерактивное меню с размерами папок
python main.py --project bpla --clean

# Удалить только кадры (frames/)
python main.py --project bpla --clean --frames

# Удалить кадры + аннотации (frames/ + annotations/)
python main.py --project bpla --clean --processed

# Удалить всё кроме raw/ (frames/ + annotations/ + dataset/ + export/)
python main.py --project bpla --clean --all-data
```

Перед удалением всегда отображается объём данных и запрашивается подтверждение `yes/no`. Папки `raw/`, `logs/` и файл `project.json` **никогда не удаляются**.

Интерактивное меню выглядит так:

```
Что удалить в проекте 'bpla'?
  1. Только кадры (frames/)                              — 6.79 ГБ
  2. Кадры + аннотации (frames/ + annotations/)           — 6.80 ГБ
  3. Все обработанные данные (frames/ + annotations/ + dataset/ + export/) — 7.21 ГБ
  4. Отмена
Выберите (1/2/3/4):
```

## Типы аугментации

Аугментации применяются к оригинальным кадрам **на их исходном разрешении**, затем результат масштабируется до целевого (640×640). Это обеспечивает корректное отображение эффектов независимо от разрешения исходного видео.

Параметры эффектов масштабируются относительно размера кадра (`h`, `w`), поэтому визуальный результат одинаков на 640×640 и на 5472×3648.

| Тип | Ключ | Описание |
|---|---|---|
| Туман | `fog` | Alpha-blending с белым overlay + сжатие контраста к серому (128) |
| Дождь | `rain` | Три слоя диагональных штрихов с перспективой, радиальный угол от центра |
| Шум | `noise` | Гауссовский шум, σ = `intensity × 50` уровней яркости |
| Размытие | `blur` | Gaussian blur с ядром от 3 до 31 px |
| Яркость | `brightness` | Изменение канала V в пространстве HSV, диапазон ×0.2 … ×2.0 |

**Параметр `intensity`** (0.0–1.0) управляет силой эффекта и задаётся единым для всех типов в шаге `--augment` (по умолчанию `0.5`).

### Дождь — детали реализации

Дождь имитирует вид с вертикально смотрящей вниз камеры БПЛА:

- В центре кадра капли вертикальны (`|`)
- К краям наклоняются до 30° (`\` слева, `/` справа)
- Три слоя с разным Gaussian blur создают глубину: крупные / средние / мелкие капли
- Фиксированное число капель (2000–2500) — одинаковая визуальная плотность при любом разрешении

## Форматы экспорта

### YOLO

Создаёт файл `export/yolo/data.yaml`:

```yaml
path: /path/to/project/dataset
train: images
val: images

nc: 1
names: ['person']
```

Файл указывает на существующую папку `dataset/images/` — изображения **не копируются**.

### COCO

Создаёт файл `export/coco/annotations.json` в стандартном формате COCO Detection:

```json
{
  "images": [
    {"id": 1, "file_name": "frame_000000.jpg", "width": 640, "height": 640}
  ],
  "annotations": [
    {
      "id": 1,
      "image_id": 1,
      "category_id": 0,
      "bbox": [120.5, 85.3, 45.2, 98.7],
      "area": 4461.24,
      "iscrowd": 0
    }
  ],
  "categories": [{"id": 0, "name": "person"}]
}
```

Координаты bbox конвертируются из нормализованного YOLO-формата `[x_c, y_c, w, h]` в абсолютный COCO-формат `[x_min, y_min, w_px, h_px]`.

## Логирование

Каждый запуск создаёт отдельный файл в папке `logs/` активного проекта:

```
projects/bpla/logs/
├── run_20260513_142300.log   # запуск 13 мая, 14:23
├── run_20260513_183015.log   # запуск 13 мая, 18:30
└── run_20260514_091200.log   # запуск 14 мая, 09:12
```

Все модули пишут в один файл через корневой логгер Python. Вывод дублируется в консоль.

**Формат строки:**
```
2026-05-13 14:23:01 [INFO]    [loader]   Processing 'video.mp4' (622 frames, step=5)
2026-05-13 14:23:05 [INFO]    [balancer] Фильтрация: 750 → 574 кадров
2026-05-13 14:23:07 [WARNING] [balancer] Папка не найдена: frames/airsim — пропускаем
```

Поиск по логу:

```bash
# Только предупреждения и ошибки
grep -E "\[(WARNING|ERROR)\]" run_20260513_142300.log

# Только записи конкретного модуля
grep "\[balancer\]" run_20260513_142300.log
```

## Примечания

### Система проектов

Каждый проект полностью изолирован — все данные, аннотации, датасет и логи хранятся внутри `projects/{name}/`. Можно вести несколько проектов параллельно без риска перемешать данные.

Метаданные в `project.json` позволяют прервать пайплайн и продолжить с того же места:

```json
{
  "name": "bpla_dataset",
  "created": "2026-05-13T15:43:52",
  "current_step": "balance",
  "start_from": "load",
  "stats": {
    "load": {"videos": 12, "frames": 5200},
    "annotated": 5200,
    "augmented_frames": 26000,
    "dataset_frames": 5557,
    "positive": 3954,
    "negative": 1603
  }
}
```

### Источники данных

Поддерживаются два источника кадров, которые объединяются в единый датасет:

- **`real`** — записи с реальной камеры (БПЛА, наземная камера)
- **`airsim`** — синтетические данные из симулятора Microsoft AirSim

Если один из источников отсутствует — пайплайн продолжает работу со вторым.

### Балансировка

По умолчанию на один позитивный кадр (с объектами) допускается до **3 негативных** (без объектов). Соотношение настраивается через `POS_NEG_RATIO` в `config.py`.

Фильтрация отбрасывает кадры, если:
- средняя яркость < 30 — кадр слишком тёмный, объекты не различимы
- дисперсия оператора Лапласа < 100.0 — кадр размытый

Оба порога настраиваются через `MIN_BRIGHTNESS` и `BLUR_THRESHOLD` в `config.py`.
