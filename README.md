# dataset_builder

Универсальный конструктор обучающей выборки для YOLO-моделей детекции объектов.  
Превращает сырые видеозаписи в готовый датасет за одну команду — от извлечения кадров до экспорта в YOLO или COCO.

---

## О проекте

### Цель и назначение

`dataset_builder` автоматизирует полный цикл подготовки данных для обучения нейронных сетей детекции объектов семейства YOLO:

1. **Извлечение кадров** из mp4-видео с заданным шагом выборки
2. **Авторазметка** с помощью предобученной модели YOLOv8
3. **Аугментация** — имитация сложных условий съёмки (туман, дождь, шум, размытие, яркость)
4. **Фильтрация и балансировка** — удаление некачественных кадров, выравнивание соотношения позитивных/негативных примеров
5. **Экспорт** в форматы YOLO (`data.yaml`) и COCO (`annotations.json`)

### Демонстрационная задача

Проект разработан для детекции людей с борта БПЛА. Источники данных:
- **real** — кадры из реальных mp4-видео (съёмка с дрона)
- **airsim** — кадры из симулятора AirSim (синтетические данные)

Два источника позволяют обучить модель на реальных условиях и расширить выборку синтетикой без дополнительной разметки.

### Три сценария использования

| Сценарий | Описание | Первый шаг |
|---|---|---|
| **Видео** | Есть mp4-файлы, всё строится с нуля | `load` |
| **Кадры** | Видео уже разбито на кадры, нужна разметка | `annotate` |
| **Датасет** | Кадры размечены, нужна балансировка и экспорт | `balance` |

### Поддерживаемые форматы экспорта

| Формат | Файл | Применение |
|---|---|---|
| **YOLO** | `export/yolo/data.yaml` | Прямое обучение YOLOv8 / YOLOv5 / YOLOv11 |
| **COCO** | `export/coco/annotations.json` | Интеграция с другими фреймворками |

## Архитектура

Проект построен как набор независимых модулей, объединённых единой точкой входа `main.py`. Каждый модуль отвечает за один шаг пайплайна.

```
main.py → project.py → loader.py → annotator.py → augmentor.py → balancer.py → exporter.py
           (изоляция)   (load)      (annotate)      (augment)      (balance)     (export)
```

### `modules/project.py` — система проектов

Управляет изоляцией данных. Каждый проект — отдельная папка с собственной структурой директорий и файлом метаданных `project.json`.

| Метод | Описание |
|---|---|
| `Project.create(name)` | Создаёт новый проект со всей структурой папок |
| `Project.load(name)` | Загружает существующий проект |
| `Project.list_all()` | Возвращает список всех проектов с метаданными |
| `Project.delete(name)` | Удаляет проект после подтверждения |
| `project.set_source(cat, key, path)` | Сохраняет путь к источнику данных в `project.json` |
| `project.get_source(cat, key)` | Возвращает путь из `data_sources` или `None` |
| `project.update_step(step)` | Обновляет `current_step` в `project.json` |
| `project.update_stats(stats)` | Дополняет поле `stats` в `project.json` |

**Категории `data_sources`:**

```
videos      → real, airsim    (папки с исходными mp4)
frames      → real, airsim    (извлечённые кадры)
annotations → real, airsim    (txt-файлы разметки YOLO)
dataset     → images, labels  (финальный датасет)
```

---

### `modules/loader.py` — извлечение кадров

**Вход:** папка с mp4-файлами (из `data_sources["videos"]`)  
**Выход:** изображения в `frames/{source}/`, запись пути в `data_sources["frames"]`

Интерактивно предлагает выбрать видеофайлы для обработки. При одном файле спрашивает подтверждение, при нескольких показывает нумерованное меню с размерами файлов.

Ключевые параметры (из `config.py`):

| Параметр | Значение | Описание |
|---|---|---|
| `FRAME_SAMPLE_RATE` | `5` | Сохранять каждый 5-й кадр (~6 fps при 30fps видео) |
| `TARGET_WIDTH` | `640` | Ширина выходных кадров |
| `TARGET_HEIGHT` | `640` | Высота выходных кадров |
| `FRAME_FORMAT` | `jpg` | Формат сохранения |

Имена файлов: `{source}_{video_stem}_frame_{index:06d}.jpg`

---

### `modules/annotator.py` — разметка кадров

**Вход:** кадры из `data_sources["frames"]`  
**Выход:** txt-файлы в `annotations/{source}/`, запись пути в `data_sources["annotations"]`

Поддерживает два режима:

| Режим | Описание |
|---|---|
| `auto` | Авторазметка через YOLOv8 (`ultralytics`). Запускает инференс, сохраняет bbox в формате YOLO txt |
| `manual` | Проверка состояния: показывает список кадров без аннотации |

В режиме `auto` интерактивно спрашивает: источник, режим, политику перезаписи существующих аннотаций.

Формат аннотации YOLO (нормализованные координаты):
```
<class_id> <x_center> <y_center> <width> <height>
```

Пустой txt-файл допустим и обозначает «негативный пример» (кадр без объектов).

---

### `modules/augmentor.py` — аугментация

**Вход:** оригинальные кадры + аннотации из папок проекта  
**Выход:** аугментированные копии рядом с оригиналами, аннотации скопированы

Для каждого оригинального кадра создаёт по одной копии каждого указанного типа аугментации. Аннотация (bbox) не изменяется — копируется из оригинала, поскольку геометрические свойства кадра не меняются. Поддерживает Unicode-пути через `np.fromfile` / `cv2.imencode`.

---

### `modules/balancer.py` — фильтрация и балансировка

**Вход:** все кадры из `frames/{source}/` (оригинальные + аугментированные)  
**Выход:** финальный датасет в `dataset/images/` и `dataset/labels/`

Три последовательных шага:

1. **Фильтрация** — отбрасывает битые, тёмные (`brightness < 30`) и размытые (`Variance of Laplacian < 100.0`) кадры. Аугментированные кадры с суффиксом `_blur` исключены из проверки резкости.
2. **Балансировка** — если негативных примеров больше чем `позитивных × 3`, случайно уменьшает негативные (undersampling). Seed фиксирован (42) для воспроизводимости.
3. **Сборка** — копирует итоговый набор в `dataset/`. Для кадров без аннотации создаёт пустой txt (YOLO-стандарт для негативного примера).

После завершения записывает в `data_sources["dataset"]` пути к `images/` и `labels/`.

---

### `modules/exporter.py` — экспорт

**Вход:** `data_sources["dataset"]["images"]` и `data_sources["dataset"]["labels"]`  
**Выход:** файлы конфигурации в `export/{format}/`

Не копирует изображения — создаёт только файлы конфигурации/аннотаций. Пути к датасету берёт из `data_sources` с fallback на `project.dataset_images/labels_dir`.

---

### `modules/logger.py` — логирование

Единая настройка для всех модулей. При старте `main.py` один раз вызывается `setup_run_log()` — создаётся файловый обработчик на корневом логгере. Все дочерние логгеры (по одному в каждом модуле) наследуют его автоматически.

## Установка

### Требования

- Python 3.8+
- OpenCV

```bash
pip install -r requirements.txt
```

Содержимое `requirements.txt`:
```
opencv-python>=4.13.0
```

Для шага **авторазметки** дополнительно:
```bash
pip install ultralytics
```

При первом запуске `ultralytics` автоматически скачает веса `yolov8n.pt` (~6 МБ).

## Быстрый старт

### Сценарий 1 — Есть видео mp4

```bash
# 1. Создать проект, выбрать "1. У меня есть видео (mp4)",
#    указать путь к папке с видео
python main.py --new-project my_project

# 2. Запустить весь пайплайн
python main.py --project my_project --all
```

Пайплайн пройдёт все 5 шагов: load → annotate → augment → balance → export.

# 1. Создать проект, выбрать "2. У меня есть кадры без разметки",

python main.py --new-project my_project

# 2. Запустить, начиная с шага разметки
python main.py --project my_project --all --from annotate
```

```

## Все команды main.py

### Управление проектами

| Команда | Описание | Пример |
|---|---|---|
| `--new-project НАЗВАНИЕ` | Создать проект (интерактивно) | `python main.py --new-project дрон` |
| `--list-projects` | Показать все проекты с ключевой статистикой | `python main.py --list-projects` |
| `--delete-project НАЗВАНИЕ` | Удалить проект (с подтверждением) | `python main.py --delete-project старый` |

---

### Запуск пайплайна

Все команды требуют `--project НАЗВАНИЕ`.

| Флаг | Описание |
|---|---|
| `--all` | Запустить весь пайплайн от шага `start_from` (задаётся при `--new-project`) |
| `--all --from ШАГ` | Запустить пайплайн начиная с указанного шага |
| `--load` | Только извлечение кадров |
| `--annotate` | Только разметка |
| `--augment` | Только аугментация |
| `--balance` | Только балансировка |
| `--export` | Только экспорт (интерактивный выбор формата) |
| `--export --format yolo` | Экспорт в формат YOLO без интерактивного вопроса |
| `--export --format coco` | Экспорт в формат COCO без интерактивного вопроса |

Допустимые значения `--from`: `load`, `annotate`, `augment`, `balance`, `export`

**Примеры:**

```bash
# Полный пайплайн
python main.py --project дрон --all

# С шага аугментации (кадры уже размечены)
python main.py --project дрон --all --from augment

# Только переэкспортировать в COCO
python main.py --project дрон --export --format coco

# Только перебалансировать датасет
python main.py --project дрон --balance
```

---

### Очистка данных проекта

`raw/`, `project.json` и `logs/` никогда не удаляются.

| Команда | Удаляет | Размер |
|---|---|---|
| `--clean` | Интерактивный выбор (показывает объём каждого уровня) | — |
| `--clean --frames` | `frames/` | только кадры |
| `--clean --processed` | `frames/` + `annotations/` | кадры и разметка |
| `--clean --all-data` | `frames/` + `annotations/` + `dataset/` + `export/` | все обработанные данные |

```bash
# Интерактивная очистка (покажет объём каждого уровня)
python main.py --project дрон --clean

# Удалить всё, кроме project.json и logs/
python main.py --project дрон --clean --all-data
```

После очистки `current_step` и связанные поля статистики в `project.json` сбрасываются.

## Система проектов

### Изоляция данных

Каждый проект — полностью независимая единица. Разные проекты могут обрабатывать разные датасеты параллельно, не влияя друг на друга. Пути к исходным данным (видео, кадры) могут указывать на любое место файловой системы — они регистрируются в `data_sources`, а не копируются в папку проекта.

### `project.json`

Файл метаданных проекта. Пример после завершения полного пайплайна:

```json
{
  "name": "my_project",
  "created": "2026-05-18T20:00:00",
  "current_step": "export",
  "start_from": "load",
  "stats": {
    "load":             {"videos": 3, "frames": 1500},
    "annotated":        1500,
    "augmented_frames": 7500,
    "dataset_frames":   6200,
    "positive":         4800,
    "negative":         1400,
    "exported_format":  "yolo",
    "exported_images":  6200
  },
  "data_sources": {
    "videos":      {"real": "C:/data/raw/real",     "airsim": null},
    "frames":      {"real": ".../frames/real",       "airsim": null},
    "annotations": {"real": ".../annotations/real",  "airsim": null},
    "dataset":     {"images": ".../dataset/images",  "labels": ".../dataset/labels"}
  }
}
```

### `data_sources` — как заполняется

Пути записываются автоматически после каждого шага:

| Шаг | Что записывается |
|---|---|
| `--new-project` | `videos.real`, `videos.airsim` |
| `load` | `frames.real`, `frames.airsim` |
| `annotate` | `annotations.real`, `annotations.airsim` |
| `balance` | `dataset.images`, `dataset.labels` |

`exporter.py` читает `dataset.images/labels` через `get_source()` с fallback на внутренние пути проекта, что позволяет использовать датасет из произвольного места файловой системы.

## Типы аугментации

Все типы применяются к оригинальным кадрам поштучно. Аннотации (bbox) не изменяются — копируются из оригинала, поскольку пространственное расположение объектов на кадре не меняется.

| Тип | Суффикс файла | Описание | Имитирует |
|---|---|---|---|
| `fog` | `_fog` | Alpha-blending с белым холстом + снижение контраста | Туман, дымку |
| `rain` | `_rain` | Три слоя капель (крупные / средние / мелкие), радиальный угол наклона относительно центра кадра, итоговый blur объектива | Дождь с перспективой камеры БПЛА |
| `noise` | `_noise` | Гауссовский шум с нулевым средним | Зернистость сенсора |
| `blur` | `_blur` | Gaussian blur (ядро масштабируется с интенсивностью) | Расфокус, тряску |
| `brightness` | `_brightness` | Случайное изменение яркости — светлее или темнее | Разные условия освещения |

**Параметр `intensity`** (0.0–1.0) управляет силой эффекта. Пайплайн запускает аугментацию с `intensity=0.5`.

Аугментация увеличивает датасет в 6 раз: 1 оригинал + 5 вариантов. Кадры с суффиксом `_blur` намеренно исключены из проверки резкости в балансировщике.

## Форматы экспорта

### YOLO — `export/yolo/data.yaml`

```yaml
path: C:/projects/my_project/dataset
train: images
val: images

nc: 1
names: ['person']
```

Указывает на папку `dataset/` проекта. Готов к использованию с командой:

```bash
yolo train data=export/yolo/data.yaml model=yolov8n.pt epochs=100
```

---

### COCO — `export/coco/annotations.json`

Полноценный файл в формате COCO Detection. Конвертирует bbox из нормализованного YOLO `[x_c, y_c, w, h]` в абсолютный COCO `[x_min, y_min, w_px, h_px]`.

```json
{
  "images": [
    {"id": 1, "file_name": "frame_000000.jpg", "width": 640, "height": 640}
  ],
  "annotations": [
    {
      "id": 1, "image_id": 1, "category_id": 0,
      "bbox": [120.5, 80.2, 45.3, 90.1],
      "area": 4087.53, "iscrowd": 0
    }
  ],
  "categories": [{"id": 0, "name": "person"}]
}
```

## Логирование

### Расположение файлов

| Файл | Путь |
|---|---|
| Лог текущего запуска | `projects/{name}/logs/run_YYYYMMDD_HHMMSS.log` |

Один файл создаётся на каждый запуск `main.py`. Все модули (loader, annotator, augmentor, balancer, exporter) пишут в один файл через наследование от корневого логгера.

### Формат записей

```
2026-05-18 21:24:28 [INFO] [loader]    Processing 'video.mp4' (622 frames, step=5)
2026-05-18 21:24:33 [INFO] [annotator] [1/125] frame_000000.jpg → найдено объектов: 8
2026-05-18 21:24:51 [INFO] [augmentor] Источник 'real' завершён: кадров=125, создано=625
2026-05-18 21:24:59 [INFO] [balancer]  Фильтрация: 750 → 574 кадров (отброшено: размытых=176)
2026-05-18 21:25:01 [INFO] [exporter]  YOLO: data.yaml → .../export/yolo/data.yaml
```

Формат: `YYYY-MM-DD HH:MM:SS [LEVEL] [module] Сообщение`

## Конфигурация

Все параметры находятся в `config.py`. Основные настройки:

```python
# Шаг выборки кадров (каждый N-й)
FRAME_SAMPLE_RATE = 5

# Размер выходных кадров (стандарт YOLO)
TARGET_WIDTH  = 640
TARGET_HEIGHT = 640

# Формат сохранения кадров
FRAME_FORMAT    = "jpg"
AUGMENT_QUALITY = 85    # JPEG-качество аугментированных кадров (0–100)

# Фильтрация некачественных кадров (balancer)
MIN_BRIGHTNESS = 30     # Минимальная средняя яркость (0–255)
BLUR_THRESHOLD = 100.0  # Минимальный Variance of Laplacian

# Балансировка классов
POS_NEG_RATIO = 3       # Негативных : позитивных (не более 3:1)

# Классы объектов
CLASS_NAMES = ['person']
```

## Известные ограничения и планы

### Текущие ограничения

| Ограничение | Описание |
|---|---|
| Нет GUI | Управление только через CLI |
| Нет train/val split | `data.yaml` указывает одну папку и для `train`, и для `val` |
| Классы заданы вручную | `CLASS_NAMES = ['person']` в `config.py` |
| Авторазметка без проверки | YOLOv8n может ошибаться; ручная корректировка не интегрирована |
